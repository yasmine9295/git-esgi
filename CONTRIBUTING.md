# Guide de contribution

## Signaler un bug

Si vous trouvez un bug, merci de créer une issue avec :

* Un titre clair
* Une description précise du problème
* Les étapes pour reproduire le bug
---

## Proposer une modification

Les contributions se font via des PR.

Conditions :

* La pull request doit être complète et prête à être revue
* Le code doit être testé et fonctionnel
* Les modifications doivent être claires et compréhensibles

Les pull requests en brouillon ou inactives peuvent être ignorées.

---

## Bonnes pratiques
* Respecter l’organisation du projet Laravel
* Tester ses modifications avant de les proposer

---

## Sécurité

Si vous découvrez une faille de sécurité, merci de ne pas créer d’issue sur le repo.
---
